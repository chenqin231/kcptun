# 文档编写与同步规范

> **触发信号**：创建/修改文档、README 更新、文档同步检查
> **来源**：dev-std §9.3 文档编写 + §3.5 文档同步 + refs/doc-standards

---

## 文档与代码同步（强制）

> **核心原则**：功能更新必须同步文档，否则不允许提交代码。

### 强制同步场景

| 变更类型 | 必须更新的文档 |
|---------|---------------|
| 新增功能 | README.md（功能列表）、使用文档 |
| 修改 API/CLI | API 文档、命令帮助 |
| 配置变更 | 配置说明文档、环境变量说明 |
| 安装方式变更 | 安装文档、README.md |
| 依赖变更 | 安装文档、CHANGELOG.md |
| 重大架构调整 | 架构文档、开发文档 |

### 文档更新工作流

```
代码修改完成 → 自测通过 → 检查是否涉及同步场景？
    ↓ 是
同步更新文档 → 提交 Git（代码 + 文档一起）→ Code Review
```

### 规范文档全链路同步

**[BLOCKING]** — 涉及命令/工作流/进度追踪变更时，必须同步更新整个文档链路。

| 层级 | 文件 | 检查项 |
|------|------|--------|
| L0 | `instructions.md` / `CLAUDE.md` | 目录结构、版本历史 |
| L1 | `development-standards.md` | 工作流、命令表 |
| 用户文档 | `README.md` | 功能特性、命令文档 |
| 变更记录 | `CHANGELOG.md` | 版本条目 |
| 版本号 | `VERSION` | SemVer 递增 |

## 禁止文档污染

### ❌ 严禁创建的文档类型

- 任务完成总结（`summary.md`、`report.md`）
- 修改说明（`changes.md`、`modifications.md`）
- 临时工作日志（`work-log.md`、`progress.md`）
- 会话记录（`session-notes.md`）
- 一次性分析（`analysis.md`、`review.md`）

**原因**：快速过时、维护负担、与 Git 提交历史冗余

### ✅ 允许创建/更新的文档

1. **核心项目文档**：README.md、CHANGELOG.md、API.md、CONTRIBUTING.md
2. **架构与设计文档**：docs/architecture.md、specs/<feature>/plan.md
3. **开发辅助文档**：docs/setup.md、docs/deployment.md

### 正确的信息记录方式

**Git 提交信息代替临时文档**：

```bash
# ✅ 正确：详细的 Git 提交信息
git commit -m "fix: 修复用户登录超时问题

问题现象：用户闲置30分钟后session过期报错
根因分析：session过期未检测
修复方案：前端增加session检测、后端统一返回401
测试验证：手动+单元+集成测试通过"

# ❌ 错误：创建 fix-session-timeout.md 文档
```

**PR Description 代替 Review 文档**
**Issue/Ticket 代替临时分析文档**

## 文档规范

### 文档类型

| 类型 | 用途 | 必需性 |
|------|------|--------|
| README | 项目介绍、安装、使用 | ✅ 必需 |
| CHANGELOG | 版本变更记录 | ✅ 必需 |
| API 文档 | API 接口说明 | 🟡 按需 |
| 设计文档 | 架构、技术选型 | 🟡 复杂项目必需 |
| 部署文档 | 部署流程、环境配置 | 🟡 推荐 |

### README 标准结构

功能特性 → 技术栈 → 安装 → 使用 → 配置 → 开发 → 测试 → 部署 → 贡献 → 许可

## 闭环管理规范

### 任务完成流程（强制）

```
完成代码 → 执行测试 → 提交 Git → 更新 tasks.md → 更新文档 → 下一任务
```

### 禁止的做法
- ❌ 完成代码后不测试
- ❌ 测试通过后不提交
- ❌ 提交后不更新 tasks.md
- ❌ 同时开启多个任务

---

## 文档清理

**定期清理**（每季度）：删除过时临时文档、归档设计文档、更新索引
**立即清理**：任务完成后删除临时文件、PR 合并后删除分析文档、功能废弃后删除相关文档

### 提交前自查清单

- [ ] 功能性变更：README.md 已更新？
- [ ] CLI 工具：命令帮助已同步？
- [ ] 配置项：配置文档已更新？
- [ ] API 变更：API 文档已同步？
- [ ] 破坏性变更：CHANGELOG.md 已记录？
- [ ] 示例代码：README.md 中的示例仍有效？
